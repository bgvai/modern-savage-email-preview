1 The source list live

54,373 legacy Bear Grylls newsletter subscribers (Mailchimp 2016–2019), imported into Supabase.

Table bg_newsletter_subscribers (status = active) · Bear Grylls Supabase project

2 The send built · gated

Script send-bg-modern-savage.mjs reads the active list and, per recipient, mints a signed one-tap Join link + one-click Unsubscribe link, renders the email, and sends via Resend from bear@beargrylls.com.

Dry-run by default · warmed resumable batches · live preflight aborts if unsubscribe links won't verify.

3 The email lands drafted

A short personal note from Bear: thanks for being on the list → Running Wild S9 → his first supplement, Modern Savage → one button: “Join the Founding Tribe” → unsubscribe in the footer. No price, no first-name merge.

They do one of three things ↓

📧 View the email draft

Path A — they join

4a One-tap join live

Click → modernsavage.co/join?e=…&t=… (their email is in the signed link — no retyping). The page verifies the token via /api/join, adds them to founding_tribe, sends the “Welcome to Modern Savage” email from bear@modernsavage.co, and shows “You're in.”

Modern Savage Supabase · dup-safe · scanner-safe (JS-gated)

Path B — they opt out

4b One-click unsubscribe live

Click (or Gmail/Yahoo one-click) → beargrylls.com/unsubscribe verifies the token and flips bg_newsletter_subscribers.status to unsubscribed, with an audit trail. Legally required, fully working.

RFC 8058 List-Unsubscribe header included

Path C — they do nothing

4c No action most recipients

They don't click either button. Nothing changes: they stay on the Bear Grylls list exactly as before, fully compliant (they were already opted in). No data is written anywhere.

The realistic majority. We can re-engage later: a follow-up to non-openers, or market Modern Savage to the whole BG list down the line (Jesse's plan).

5 The outcome live

Joiners are now on the Modern Savage founding_tribe list and receive launch comms from Bear. Campaign conversion = cross-reference the send list against join timestamps.
